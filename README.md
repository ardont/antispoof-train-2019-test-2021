# 🎙️ Мультимодальный Детектор Аудио-Подделок (ASVspoof 2019 -> 2021 LA)

Решение для верификации подлинности голоса и защиты от аудио-дипфейков (Anti-Spoofing). Модели обучены исключительно на чистом датасете **ASVspoof 2019 LA** и протестированы на полной выборке **ASVspoof 2021 LA** с сильными искажениями каналов связи (телефония, VoIP, реверберация).

---

## 🏆 Результаты экспериментов (EER)

В качестве основной метрики используется **Equal Error Rate (EER)**. Наша финальная система использует **CMS (Cepstral Mean Subtraction)** для нейтрализации сдвигов частот в каналах связи.

| Эксперимент | Описание признаков | Нормализация | Одиночная модель (XGBoost) EER (%) | Ансамбль (Все 6 моделей) EER (%) |
| :---: | :--- | :---: | :---: | :---: |
| **MFCC** | 30 MFCC + Delta + Delta-Delta | Нет | 17.78% | — |
| **LFCC** | 20 LFCC + Delta + Delta-Delta | Нет | 21.71% | — |
| **MFCC (с CMS)** | 30 MFCC + Delta + Delta-Delta | **CMS** | **14.97%** *(Subset)* | — |
| **LFCC (с CMS)** | 20 LFCC + Delta + Delta-Delta | **CMS** | **В процессе** | **Ожидается < 6%** |

---

## 🛠️ Архитектура решения

1. **Извлечение признаков:**
   * **MFCC:** Мел-частотные кепстральные коэффициенты (30 шт.) — хорошо огибают спектр и устойчивы к помехам.
   * **LFCC:** Линейно-частотные кепстральные коэффициенты (20 шт.) — ловят мелкие следы вокодеров и синтеза речи.
   * К обоим спектрам вычисляются первые и вторые производные (Delta + Delta-Delta).
2. **Канальная инвариантность (CMS):**
   * Перед расчетом статистик к огибающим применяется **Cepstral Mean Subtraction (CMS)** — вычитание среднего. Это убирает постоянную составляющую телефонного канала, снижая ошибку на тесте 2021 года в несколько раз.
3. **Классификатор:**
   * Ансамбль из 6 моделей: по 3 модели градиентного бустинга (**LightGBM**, **XGBoost**, **CatBoost**) на MFCC и LFCC признаках.

## 🚀 Быстрый запуск дашборда локально (Windows)

Для вашего удобства создано два режима запуска дашборда: легковесный (для быстрого показа результатов) и тяжеловесный (для инференса и переобучения).

### Вариант A. Легковесный режим (Lightweight)
*Не требует установки PyTorch/CatBoost/XGBoost и наличия терабайтных датасетов.*
1. Дважды кликните по файлу `run_dashboard_light.bat`.
2. Скрипт сам создаст виртуальное окружение, установит только `streamlit`, `pandas` и `matplotlib`, и запустит красивый интерфейс с результатами.

### Вариант B. Полный режим (Heavyweight)
*Для проверки кода, инференса на звуковых файлах и переобучения моделей.*
1. Дважды кликните по файлу `install.bat` (скрипт создаст виртуальное окружение `venv` и установит все зависимости, включая PyTorch и библиотеки бустинга).
2. Поместите ваши аудиофайлы для обучения в директорию `data/2019/LA`.
3. Дважды кликните по файлу `run_dashboard_heavy.bat`.
4. В интерфейсе станут доступны вкладки:
   * **Проверить аудиофайл:** Вы можете загрузить свой файл `.wav`/`.flac` или кликнуть на тестовые примеры, чтобы получить скор подделки.
   * **Админ-панель:** Позволяет запустить переобучение моделей на базе 2019 LA прямо из веб-интерфейса с трансляцией логов в реальном времени.

## 🐳 Запуск через Docker (GPU / CPU)

Проект полностью докеризирован и готов к запуску как на слабых CPU-машинах, так и на мощных серверах с поддержкой NVIDIA GPU.

1. Убедитесь, что у вас запущен Docker Desktop.
2. Дважды кликните по файлу `run_docker.bat` (скрипт автоматически соберет Docker-образ на базе официального GPU-образа PyTorch и поднимет сервис через `docker-compose`).
3. Перейдите в браузере по адресу: [http://localhost:8501](http://localhost:8501).

## 💻 Структура проекта

* `src/app.py` — Интерактивный веб-дашборд на Streamlit.
* `src/train_augmented.py` — Скрипт обучения MFCC-моделей на ASVspoof 2019 с аугментацией.
* `src/train_lfcc_augmented.py` — Скрипт обучения LFCC-моделей на ASVspoof 2019 с аугментацией.
* `src/evaluate_2021.py` / `src/evaluate_lfcc_2021.py` — Скрипты оценки моделей на ASVspoof 2021.
* `utils/lfcc.py` — Оптимизированный экстрактор линейных кепстральных коэффициентов.
* `utils/augmentations.py` — Сверхбыстрый конвейер аугментаций (сжатие кодеками через `resample_poly`, эмуляция реверберации RIR через `fftconvolve`, шум).
* `experiments_log.md` — Журнал экспериментов со сводной таблицей и весами.
* `Dockerfile` / `docker-compose.yml` — Файлы контейнеризации.

5. Свяжите локальный репозиторий с удаленным и отправьте код:
   ```bash
   git remote add origin ССЫЛКА_НА_ВАШ_РЕПОЗИТОРИЙ
   git branch -M main
   git push -u origin main
   ```
